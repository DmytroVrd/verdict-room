# Verdict Room — business case

> Джерело правди для pitch / README / відео / слайдів. Англ. формулювання —
> готові до копіювання в подачу; укр. коментарі — для Дмитра.

## One-liner
**Verdict Room turns weeks of vendor due diligence into a 3-minute
multi-agent debate — and gives you a defensible verdict, not just an opinion.**

## Problem (з цифрами)
Choosing a B2B software vendor (CRM, workspace, analytics tool…) means one
person juggling pricing, security, GDPR, migration risk and alternatives.
- Typically **20–40 hours of manual research** spread over **2–4 weeks**.
- Output is **one person's biased summary** — no adversarial check, weak paper trail.
- Procurement/ops teams do this **dozens of times a year**.

## Target user
Procurement / operations / IT lead at a **50–500-person company** who repeatedly
evaluates SaaS tools and vendors. (Also: founders, consultants doing vendor picks.)

## Solution
Drop a case into a Band room → six specialized agents run a structured debate:
Researcher gathers sourced facts, Scout finds alternatives, Critic attacks,
Advocate defends, a Compliance specialist is **pulled in mid-debate** when data
risk surfaces, and an Arbiter issues a scored verdict
(BUY / BUY_WITH_CONDITIONS / AVOID) with conditions and a dissent.
**Like an investment-committee review, but in minutes.**

## Why multi-agent / why Band (не один агент)
Real decisions need **adversarial review**, not a single confident answer.
One LLM gives one biased take. A *room* of agents — advocate vs critic, plus an
independent compliance voice — produces a balanced verdict with explicit dissent.
Band is what makes that possible: agents **hand off tasks, share context,
recruit each other** in a shared room. This only became feasible with this
generation of collaborating agents.

## Originality (назвати вголос у відео)
- **Dynamic recruitment**: the Arbiter pulls a Compliance agent INTO the live
  debate the moment a data-privacy risk appears — agents recruiting agents.
- **Cross-model courtroom**: debaters run on different models/providers
  (AI/ML API + Featherless), so it's a genuine multi-perspective panel,
  not one model talking to itself.

## Monetization
- **Per-verdict**: pay per deep vendor analysis (one-off buyers / consultants).
- **SaaS per-seat**: procurement teams running many evaluations a month.

## Market (груба оцінка, подавати як rough)
Procurement software ≈ **$7–9B** and growing; AI-assisted procurement is the
fast-growing slice. Even a thin layer on repeated vendor decisions is meaningful.

## Demo case (основний для відео)
"Should a 50-person EU software company adopt Notion as its main workspace vs
Confluence and Slite? Prioritize cost, migration, security, GDPR."
→ triggers the GDPR/data-privacy path → Compliance gets recruited live → verdict.
(Bonus second case for universality: choosing a laptop — B2C.)
